Globe Implementation Approach

Which rendering strategy for the rotating tech globe?

A

Three.js + React Three Fiber

True WebGL 3D globe. Icons as sprites pinned to sphere surface. Closest match to the pszostak.pl reference — glowing mesh, proper depth, smooth inertia drag. Adds ~400KB (gzipped) to bundle.

Pros

  • Most realistic 3D effect
  • Proper perspective & depth sort
  • Inertia on drag feels natural

Cons

  • Large dependency (~400KB gz)
  • More complex setup
B

CSS 3D + Trigonometry

No new deps. Icons positioned using sin/cos math, rendered as DOM elements with CSS perspective + transform: translateZ. Depth-fade via opacity. Uses Framer Motion for drag + rotation.

Pros

  • Zero new dependencies
  • Full React/Tailwind styling on icons
  • Easy hover state & tooltips

Cons

  • No real depth sorting (icons overlap wrong at edges)
  • Less immersive than WebGL
C

Canvas 2D + Sphere Projection

Draw everything on an HTML Canvas. Sphere grid lines + icons drawn as canvas images. Handles depth-sort correctly. No new deps, good perf.

Pros

  • Zero new deps
  • Correct depth sorting
  • Good performance

Cons

  • Harder to style icons (canvas draws, not DOM)
  • Tooltip/hover needs manual hit-testing
  • Less crisp on retina screens
